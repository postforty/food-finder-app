# 푸드 파인더 앱 프로젝트 분석 (Project Analysis)

## 1. 기술 스택
- **프레임워크**: Next.js 16.0.8 (App Router)
- **언어**: TypeScript
- **UI 라이브러리**: React 19.2.1
- **스타일링**: Tailwind CSS v4, PostCSS
- **백엔드 / 데이터베이스**: Firebase v12.6.0 (Client SDK)
- **PWA 지원**: next-pwa

## 2. 프로젝트 구조
이 프로젝트는 TypeScript와 함께 표준 Next.js App Router 구조를 따릅니다.

### 핵심 디렉토리
- `src/app`: 애플리케이션 라우트 및 페이지를 포함합니다.
  - `/`: 메인 랜딩 페이지 (`page.tsx`)
  - `/admin`: 관리자 대시보드 또는 관리 영역
  - `/favorites`: 사용자 즐겨찾기 목록
  - `/restaurants`: 맛집 목록 또는 상세 정보
- `src/components`: 재사용 가능한 UI 컴포넌트.
  - `Header.tsx`: 메인 내비게이션 헤더
  - `providers/`: 컨텍스트 프로바이더 (주로 클라이언트 사이드 로직용)
- `src/context`: React Context 정의 (전역 상태 관리).
- `src/lib`: 유틸리티 라이브러리.
  - `firebase/`: Firebase 설정 및 초기화.

### 설정 파일
- `next.config.ts`: Next.js 설정
- `tailwind.config.js` / `postcss.config.mjs`: 스타일링 설정 (Tailwind v4는 CSS 변수를 광범위하게 사용하거나 설정이 더 단순할 수 있음).
- `firebase.json` / `.firebaserc`: (배포를 위해 암시/예상됨, 현재까지는 주로 클라이언트 사이드 사용만 보임).

## 3. 식별된 주요 기능
1.  **맛집 탐색 (로그인 불필요)**: 사용자는 인증 없이 맛집을 둘러보고 검색할 수 있습니다.
2.  **외부 지도 통합**: 각 맛집 항목에는 네이버 지도로 연결되는 직접 링크가 포함됩니다.
3.  **내부 지도 없음**: 앱은 내부적으로 지도를 렌더링하지 않으며, 전적으로 외부 지도 서비스에 의존합니다.
4.  **관리자 인터페이스**: (추정) 관리자만이 맛집 데이터를 등록/관리할 수 있습니다.
5.  **즐겨찾기 시스템**: (보류 중) 조회에 로그인이 필요하지 않다면, 즐겨찾기는 로컬 스토리지 또는 이 기능을 위해서만 로그인을 요구할 가능성이 큽니다.
6.  **PWA**: 프로그레시브 웹 앱으로 설치 가능하도록 설정되어 있습니다.

## 4. 관찰 및 권장 사항
-   **데이터 구조 업데이트**: 맛집 데이터 모델은 네이버 지도 링크를 저장할 `mapUrl` 필드가 필요합니다.
-   **사용자 흐름**:
    -   사용자 페이지 방문 -> 탐색/검색 -> 맛집 클릭 -> 새 탭에서 네이버 지도 열기.
-   **스타일링**: 프로젝트는 Tailwind CSS v4를 사용합니다. 모든 새 컴포넌트가 v4 규칙에 맞는 유틸리티 클래스를 사용하도록 해야 합니다.
-   **상태 관리**: React Context (`src/context`) 사용 및 데이터 영속성을 위해 Firebase를 사용할 것으로 보입니다.
-   **데이터 페칭**: Firebase SDK 또는 Server Actions를 사용한 클라이언트 사이드 페칭일 가능성이 높습니다.
