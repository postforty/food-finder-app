# 현재 구현 현황

## 1. 랜딩 페이지 (`src/app/page.tsx`)
홈 페이지는 다음과 같은 기능을 포함한 완전한 반응형 랜딩 페이지입니다:
-   **히어로 섹션**: 애니메이션 효과가 적용된 타이틀, 검색바(현재는 UI만 존재), 빠른 통계 표시.
-   **카테고리**: 6가지 음식 카테고리(한식, 중식, 일식, 양식, 카페, 디저트)로 구성된 그리드.
-   **추천 맛집**: 평점과 태그가 포함된 인기 맛집 목업(Mock) 리스트.
-   **푸터**: 서비스, 회사, 법적 고지 페이지 링크가 포함된 표준 푸터.

## 2. 인증
-   **컨텍스트**: `AuthContext`가 `@/context/AuthContext`에 설정되어 있습니다.
-   **사용**: 랜딩 페이지에서 `user` 객체 여부를 확인하여 "빠른 통계" 섹션을 조건부로 렌더링합니다.

## 3. 목업 데이터 (Mock Data)
현재 애플리케이션은 다음 항목들에 대해 컴포넌트 파일 내에 하드코딩된 목업 데이터를 사용하고 있습니다:
-   **카테고리**: ID, 이름, 이모지, 색상 그라디언트.
-   **맛집**: ID, 이름, 카테고리, 평점, 리뷰 수, 이미지(이모지), 거리, 태그.

## 4. 구현 예정 사항 (TODOs)
-   **검색 기능**: 검색 양식은 존재하지만, 현재는 콘솔에 로그만 출력됩니다.
-   **실제 데이터 연동**: 목업 배열을 Firebase 데이터 페칭으로 대체해야 합니다.
-   **내비게이션**: `/restaurants`, `/favorites` 링크가 있지만, 실제 페이지 구현과 확인이 필요합니다.
-   **즐겨찾기**: 맛집 카드에 버튼은 존재하지만, 기능은 UI 전용(호버 효과)으로만 작동합니다.

## 5. UI/UX
-   **디자인 시스템**: 테마 지정을 위해 CSS 변수(`--background`, `--foreground` 등)를 사용하고 있어, 접근성 높은 다크 모드 지원을 암시합니다.
-   **애니메이션**: 바닐라 CSS 애니메이션 또는 Tailwind 확장 클래스(`animate-fadeIn`, `animate-scaleIn`, `animate-float`)를 사용합니다.
